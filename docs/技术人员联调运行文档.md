## 2. 联调方式选择

### 方式 A：远程（你在外地、技术员在有设备处）— 推荐

```
你（浏览器）  →  https://kneejoy.onrender.com  →  云端 API  →  ESP32（技术员侧 Wi-Fi）
```

- **你**：打开云地址，患者登录，点「开始治疗」
- **技术员**：ESP32 `config.h` 指向 `kneejoy.onrender.com`，串口看 `START`
- **缩回按钮**：云端 HTTPS **无法**直连 `http://192.168.x.x`，缩回需方式 B

### 方式 B：局域网直连 ESP32 HTTP（缩回 / status / therapy 全功能）

```
电脑/手机（npm run start）  →  http://<ESP32_IP>/api/*  （同一 Wi-Fi）
```

- 必须在 **同一局域网**
- 在 App「硬件联调」填 ESP32 IP，或 `.env.local` 设置 `VITE_DEVICE_IP`

## 4. 技术员：ESP32 准备

### 4.1 配置文件

`firmware/esp32-kneejoy/KneeJoy_Device/config.h`（由 `config.example.h` 复制）

**远程云端联调：**

```c
#define API_HOST "kneejoy.onrender.com"
#define API_USE_TLS 1
#define DEVICE_ID "KJ-DEMO-001"
#define DEVICE_TOKEN "kneejoy-demo-token-2026"
```

**局域网直连联调：**

```c
#define API_HOST "192.168.x.x"   // 运行 npm run start 的电脑 IP
#define API_PORT 3001
#define API_USE_TLS 0
```

### 4.2 上传固件

Arduino IDE → `KneeJoy_Device.ino` → 板型 ESP32 Dev Module → 115200

成功标志：`[云端] Ping 成功` 或 `[Wi-Fi] 已连接`

## 5. 命令行验证（技术员先做）

### 5.1 云端通道

```bash
curl https://kneejoy.onrender.com/api/v1/device/ping \
  -H "X-Device-Id: KJ-DEMO-001" \
  -H "X-Device-Token: kneejoy-demo-token-2026"
```

### 5.2 ESP32 局域网 HTTP（替换 IP）

```bash
npm run verify:device -- 192.168.1.100
```

应输出 `GET /api/device` 与 `GET /api/status`。

### 5.3 缩回接口单测

```bash
# 开始双侧缩回
curl -X POST "http://192.168.1.100/api/motor?side=all&action=retract"

# 停止缩回
curl -X POST "http://192.168.1.100/api/motor?side=all&action=stop"
```

## 6. App 内 UI 联调步骤

1. 打开 http://localhost:3000 ，患者登录
2. **设置** → **Wi-Fi 连接**（或蓝牙连接后可见 UI，真 API 需 Wi-Fi + IP）
3. 进入 **理疗控制面板**
4. 调参数 → 点 **开始物理康复治疗**
5. 治疗结束 → 点 **结束治疗**
6. 点 **缩回推杆**（或 左侧 / 右侧）
7. 推杆到位后点 **停止**

**硬件联调浮窗（可选）：**

- 右下角「硬件联调」→ 填 ESP32 IP → 连接 → 1Hz 轮询 → 下发治疗 / 缩回

## 7. 联调检查清单

### 固件单机

- [ ] 串口 115200 可见 banner

- [ ] `GET http://<IP>/api/status` 力/温度字段刷新

## 8. 常见问题

| 现象 | 处理 |
|------|------|
| 缩回按钮灰色 | 先结束治疗；检查是否急停 |
| 点了没反应 | 需 Wi-Fi 直连 + 硬件联调填 IP；蓝牙仅为 UI 演示 |
| 云端打不开 | 免费 Render 休眠，等 30～60 秒刷新 |
| 手机访问不了电脑 IP | 确认同一 Wi-Fi；防火墙放行 3000/3001 |
| `verify:device` 失败 | 确认 ESP32 IP、CORS、设备 HTTP 服务已启动 |
